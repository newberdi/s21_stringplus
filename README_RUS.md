# s21_string (групповой проект)

Собственная реализация ключевых компонентов стандартной библиотеки C `string.h`, включая функции форматированного ввода/вывода и дополнительные инструменты обработки строк.

Проект посвящён изучению работы со строками, памятью, вариативными аргументами, алгоритмами парсинга и внутренним механизмам стандартной библиотеки языка C.

## Обзор

Библиотека содержит собственные реализации основных функций работы со строками и памятью без использования оригинальных реализаций `string.h`.

Дополнительно реализованы аналоги функций:

* `sprintf`
* `sscanf`

а также набор вспомогательных функций для обработки строк.

Основная задача проекта — максимально точно воспроизвести поведение стандартной библиотеки C и проверить корректность работы с помощью автоматизированных тестов.

## Реализованные функции

### Работа с памятью

* `memchr`
* `memcmp`
* `memcpy`
* `memset`

### Работа со строками

* `strncat`
* `strchr`
* `strncmp`
* `strncpy`
* `strcspn`
* `strerror`
* `strlen`
* `strpbrk`
* `strrchr`
* `strstr`
* `strtok`

### Форматированный вывод

Собственная реализация:

```c
int s21_sprintf(char *str, const char *format, ...);
```

Поддерживает:

* целочисленные типы;
* числа с плавающей точкой;
* строки и символы;
* ширину поля;
* точность вывода;
* модификаторы длины;
* флаги форматирования.

### Форматированный ввод

Собственная реализация:

```c
int s21_sscanf(const char *str, const char *format, ...);
```

Поддерживает:

* преобразование числовых типов;
* обработку строк;
* работу с числами с плавающей точкой;
* модификаторы ширины;
* модификаторы длины;
* управление присваиванием значений.

### Дополнительные функции

* `s21_to_upper`
* `s21_to_lower`
* `s21_insert`
* `s21_trim`

## Технические детали

* Язык: C11
* Компилятор: GCC
* Система сборки: Makefile
* Фреймворк тестирования: Check
* Стиль кода: Google C Style
* Подход: структурное программирование

## Тестирование

Проект включает:

* модульные тесты для всех функций;
* сравнение результатов со стандартной библиотекой C;
* анализ покрытия кода через `gcov`;
* автоматизированную сборку и запуск тестов через Makefile.

## Сборка

Собрать проект:

```bash
make
```

Собрать библиотеку:

```bash
make s21_string.a
```

Запустить тесты:

```bash
make test
```

Сформировать отчёт покрытия:

```bash
make gcov_report
```
